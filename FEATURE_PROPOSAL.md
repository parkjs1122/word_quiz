# Word Quiz 기능 개선 기획안

> 작성일: 2026-02-23
> 대상: https://jswordquiz.vercel.app

---

## 현재 상태 요약

| 항목 | 현황 |
|------|------|
| 인증 | 이메일/비밀번호 기반 회원가입·로그인 |
| 단어 관리 | .txt 업로드, 폴더 분류, 검색/필터, 벌크 작업 |
| 퀴즈 | 단어 → 뜻 확인 방식, 외움/모름 이진 판정, 세션 저장 |
| 대시보드 | 전체/외운/안외운 단어 수, 암기율 도넛 차트 |
| 기타 | 다크모드, TTS, 반응형, 단어 내보내기 |

## 1. 학습 효과 강화

### 1-1. 퀴즈 모드 다양화

**현재 문제점:** 퀴즈가 "단어 보여주기 → 3초 후 뜻 공개 → O/X 판정" 한 가지뿐이라 반복 시 학습 효과가 떨어짐

| 모드 | 설명 | 우선순위 |
|------|------|----------|
| **뜻 → 단어** | 뜻을 먼저 보여주고 단어를 맞히는 역방향 퀴즈 | 🔴 높음 |
| **4지선다** | 뜻 4개 중 정답 고르기 (같은 폴더 단어로 보기 생성) | 🔴 높음 |
| **직접 입력(타이핑)** | 뜻을 보고 단어를 직접 타이핑하여 스펠링까지 학습 | 🟡 중간 |
| **받아쓰기** | TTS로 발음 듣고 단어를 입력하는 리스닝 퀴즈 | 🟢 낮음 |

### 1-2. 간격 반복 학습 (Spaced Repetition)

**현재 문제점:** 암기 상태가 `memorized: boolean`으로만 관리되어 "외웠다/못외웠다"의 이진 분류뿐, 망각 곡선을 반영하지 못함

**개선안:**
- 단어별 `level` (0~5) 및 `nextReviewAt` 필드 추가
- 정답 시 레벨 상승 → 복습 간격 증가 (1일 → 3일 → 7일 → 14일 → 30일)
- 오답 시 레벨 하락 → 복습 간격 단축
- 대시보드에 "오늘 복습할 단어 N개" 표시
- 우선순위: 🔴 높음

### 1-3. 오답 노트

**현재 문제점:** 퀴즈에서 틀린 단어를 따로 모아볼 수 없음

**개선안:**
- 퀴즈 완료 시 틀린 단어 목록 제공
- "틀린 단어만 다시 풀기" 버튼
- 마이페이지에서 최근 오답 단어 확인 가능
- 우선순위: 🔴 높음

## 2. 단어 관리 개선

### 2-1. 다양한 입력 방식 지원

**현재 문제점:** .txt 탭 구분 파일만 업로드 가능 — 진입 장벽이 높음

| 방식 | 설명 | 우선순위 |
|------|------|----------|
| **수동 추가 폼** | 단어/뜻을 직접 입력하는 인라인 폼 (테이블 상단에 추가 행) | 🔴 높음 |
| **CSV/Excel 지원** | .csv, .xlsx 파일 업로드 지원 | 🟡 중간 |
| **붙여넣기 입력** | 텍스트 영역에 여러 줄 붙여넣기로 한번에 등록 | 🟡 중간 |

### 2-2. 단어 부가 정보

**현재 문제점:** 단어에 `word`와 `meaning`만 있어 학습 맥락이 부족

**개선안:**
- 예문(example sentence) 필드 추가
- 품사(part of speech) 태그 추가
- 메모/노트 필드 추가 (개인 기억법 기록)
- 우선순위: 🟡 중간

### 2-3. 폴더 기능 강화

**현재 문제점:** 폴더가 1단계 구조, 정렬·색상 구분 없음

**개선안:**
- 폴더별 색상 지정 (시각적 구분)
- 폴더 순서 드래그 정렬
- 폴더 내 단어 순서 커스텀 정렬 (이름순, 등록순, 암기율순)
- 우선순위: 🟢 낮음

## 3. 대시보드 및 통계 강화

### 3-1. 학습 기록 및 추이 그래프

**현재 문제점:** 대시보드에 현재 시점의 암기율만 표시 — 학습 추이를 확인할 수 없음

**개선안:**
- 일별/주별 학습 기록 저장 (퀴즈 횟수, 정답률, 학습 단어 수)
- 꺾은선 그래프로 암기율 변화 추이 시각화
- 주간 학습 요약 (이번 주 학습한 단어 수, 새로 외운 단어)
- DB에 `QuizHistory` 모델 추가 필요
- 우선순위: 🟡 중간

### 3-2. 폴더별 통계

**현재 문제점:** 대시보드 통계가 전체 단어 기준 — 폴더별 진행도를 한눈에 보기 어려움

**개선안:**
- 폴더별 암기율 프로그레스 바 목록
- 폴더별 미니 도넛 차트
- 가장 학습이 필요한 폴더 하이라이트
- 우선순위: 🟡 중간

### 3-3. 학습 연속 기록 (스트릭)

**현재 문제점:** 매일 학습할 동기가 부족

**개선안:**
- 연속 학습일 카운트 (🔥 3일 연속!)
- 캘린더 히트맵 (GitHub 잔디처럼 학습 활동 시각화)
- 일일 학습 목표 설정 (예: 하루 20단어)
- 우선순위: 🟡 중간

## 4. UI/UX 개선

### 4-1. 퀴즈 화면 UX

**현재 문제점:**
- 3초 자동 카운트다운이 고정 — 사용자가 준비되기 전에 뜻이 공개될 수 있음
- 키보드 조작 불가

**개선안:**
- 카운트다운 대신 **"뜻 보기" 탭/클릭 버튼**으로 사용자가 직접 제어 (옵션 선택 가능)
- 키보드 단축키 지원: `Space`=뜻 보기, `←`=모름, `→`=외움
- 스와이프 제스처 지원 (모바일): 오른쪽=외움, 왼쪽=모름
- 카드 넘기는 애니메이션 추가 (flip 또는 slide)
- 우선순위: 🔴 높음

### 4-2. 마이페이지 레이아웃 개선

**현재 문제점:**
- 단어 업로드 영역이 항상 상단에 노출되어 공간 차지
- 모바일에서 폴더 사이드바 접근이 불편할 수 있음

**개선안:**
- 단어 업로드를 모달 또는 접을 수 있는 섹션으로 변경
- 모바일에서 폴더를 드롭다운 또는 하단 시트로 표시
- 단어 목록에 "최근 추가" 배지 표시
- 우선순위: 🟡 중간

### 4-3. 온보딩 가이드

**현재 문제점:** 처음 가입한 사용자가 어디서부터 시작해야 할지 알기 어려움

**개선안:**
- 첫 로그인 시 단계별 안내 (1. 단어 추가 → 2. 폴더 정리 → 3. 퀴즈 시작)
- 비어있는 대시보드에 시작 가이드 카드 표시
- 샘플 단어 셋 제공 (선택적 자동 등록)
- 우선순위: 🟢 낮음

### 4-4. 토스트 알림

**현재 문제점:** 단어 저장, 삭제 등 액션 피드백이 인라인 메시지로만 표시되어 눈에 잘 띄지 않을 수 있음

**개선안:**
- 하단 토스트 알림으로 액션 결과 피드백 (저장 완료, 삭제 완료, 에러 등)
- 자동 소멸 (3초)
- 우선순위: 🟢 낮음

## 5. 접근성 및 편의 기능

### 5-1. PWA 지원

**현재 문제점:** 모바일에서 매번 브라우저로 접속해야 함

**개선안:**
- Service Worker + Web App Manifest 추가
- 홈 화면 추가 가능한 PWA 구성
- 오프라인 시 마지막 동기화 단어로 퀴즈 가능
- 우선순위: 🟡 중간

### 5-2. 단어 공유/가져오기

**현재 문제점:** 사용자 간 단어 목록 공유 불가

**개선안:**
- 폴더를 공유 링크로 내보내기 (읽기 전용 페이지)
- 공유 링크에서 "내 단어장에 추가" 버튼
- 우선순위: 🟢 낮음

### 5-3. 비밀번호 재설정

**현재 문제점:** 비밀번호 분실 시 복구 수단 없음

**개선안:**
- 이메일 기반 비밀번호 재설정 플로우
- 또는 소셜 로그인 추가 (Google, GitHub)
- 우선순위: 🟡 중간

---

## 6. 기술적 개선

### 6-1. 검색 성능 최적화

**현재 문제점:** 클라이언트 사이드에서 필터링 중 — 단어 수가 많아지면 성능 저하 우려

**개선안:**
- 서버 사이드 검색 + 디바운스 적용
- Prisma full-text search 활용
- 우선순위: 🟢 낮음 (현재 규모에서는 문제 없음)

### 6-2. 에러 처리 고도화

**현재 문제점:** Server Action 에러가 일부 일반적인 메시지만 반환

**개선안:**
- 글로벌 에러 바운더리 추가
- 네트워크 에러 시 재시도 안내
- 로딩 스켈레톤 UI 추가
- 우선순위: 🟢 낮음

## 권장 구현 순서 (로드맵)

### Phase 1 — 학습 핵심 강화
> 수동 단어 추가 폼 → 퀴즈 UX 개선 → 퀴즈 모드 다양화 → 오답 노트

사용자의 핵심 플로우인 "단어 등록 → 퀴즈 학습"을 먼저 강화합니다.

### Phase 2 — 동기부여 및 추적
> 간격 반복 학습 → 학습 기록/그래프 → 학습 스트릭 → 폴더별 통계

학습 지속성을 높이는 기능을 추가합니다.

### Phase 3 — 편의 및 확장
> PWA → 마이페이지 개선 → 비밀번호 재설정 → 단어 부가 정보

접근성과 편의성을 개선합니다.

### Phase 4 — 부가 기능
> 토스트 알림 → 온보딩 → 단어 공유 → 폴더 강화

완성도를 높이는 부가 기능을 추가합니다.
